The difference between values from converting to a number or not do it is noticeable. When your not converting to a number, then javascript will read it as a string. That means it will write itself out as text like 5 + 5 = 55 and not 10. Only when you multiply then javascript will notice its a number when you use the multiplier mark (*). To get every mathematical operators work perfectly you can solve the problem by using parseInt that will convert the value to an integer and write out as a mathematical result. If it happends that you want javascript to write out decimal numbers? you can instead of parseInt use parseFloat.
